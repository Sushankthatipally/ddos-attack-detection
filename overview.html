Federated DDoS IDS - Formula Sheet

Readable version for study and print (grouped by pipeline stage).

Quick Legend: x = sample vector, mu = centroid, theta = threshold, lambda = anomaly buffer, TP/TN/FP/FN = confusion matrix terms, y_hat = predicted class.

A) Preprocessing

F1. Missing value imputation

\[x_{ij}=\bar{x}_j\quad(\text{if }x_{ij}\text{ is NaN})\]
  • x_ij: value at sample i, feature j
  • mean_j: mean of feature j

Use: Replace missing numeric values before scaling.

File: client/preprocess.py

F2. Constant feature removal

\[\mathrm{Var}(X_j)=0\Rightarrow \text{drop feature }j\]
  • X_j: all values of feature j
  • Var(. ): variance

Use: Remove non-informative columns.

File: client/preprocess.py

F3. Standardization (z-score)

\[z_{ij}=\frac{x_{ij}-\mu_j}{\sigma_j}\]
  • mu_j: mean of feature j
  • sigma_j: std deviation of feature j
  • z_ij: scaled value used for clustering

Use: Bring all features to comparable scale.

File: client/preprocess.py

B) Distance Metrics

F4. Euclidean distance

\[D_e(x,\mu)=\sqrt{\sum_j(x_j-\mu_j)^2}\]
  • x: sample vector
  • mu: centroid vector
  • j: feature index

Use: Straight-line distance in feature space.

File: client/clustering.py

F5. Manhattan distance

\[D_m(x,\mu)=\sum_j\left|x_j-\mu_j\right|\]
  • |.|: absolute value
  • x_j, mu_j: feature values at index j

Use: L1 distance (cityblock metric).

File: client/clustering.py

F6. Minkowski distance

\[D_p(x,\mu)=\left(\sum_j\left|x_j-\mu_j\right|^p\right)^{1/p}\]
  • p: Minkowski order (p >= 1)
  • p=2 gives Euclidean, p=1 gives Manhattan

Use: Flexible generalized distance metric.

File: client/clustering.py, config.py

F7. Cosine distance

\[D_{\cos}(x,\mu)=1-\frac{x\cdot\mu}{\lVert x\rVert_2\lVert\mu\rVert_2}\]
  • x dot mu: dot product
  • ||x||, ||mu||: L2 norm (vector length)

Use: Measures angle difference (direction similarity).

File: client/clustering.py

C) SCC Cluster Formation

F8. Nearest cluster selection

\[k^*=\arg\min_k D(x,\mu_k),\quad d_{\min}=D(x,\mu_{k^*})\]
  • k*: nearest centroid index
  • d_min: minimum distance for current sample

Use: Decide candidate cluster for assignment.

File: client/clustering.py

F9. Cluster creation rule

\[d_{\min}\le \theta\Rightarrow \text{assign};\ \text{else create new cluster}\]
  • theta: cluster threshold (predefined hyperparameter)

Use: Controls number of clusters formed.

File: client/clustering.py, config.py, app.py

F10. Exact centroid update

\[\mu_k^{\text{new}}=\frac{n_k\mu_k^{\text{old}}+x}{n_k+1}\]
  • n_k: current sample count in cluster k
  • mu_k_old, mu_k_new: old/new centroid

Use: Running mean update after assignment.

File: client/clustering.py

F11. Distance-space transform

\[z_i=[D(x_i,\mu_1),D(x_i,\mu_2),\ldots,D(x_i,\mu_C)]\]
  • z_i: distance vector for sample i
  • C: number of clusters

Use: Returns sample distances to all centroids.

File: client/clustering.py

D) Thresholds and Detection

F12. Global anomaly threshold

\[\theta_{\text{anomaly}}=\mu_D+\lambda\sigma_D\]
  • D: all minimum centroid distances from training samples
  • mu_D: mean of D
  • sigma_D: std of D
  • lambda: anomaly buffer

Use: Default detection threshold after training.

File: client/clustering.py, config.py

F13. Adaptive cluster threshold

\[\theta_k=\mu_{D_k}+\lambda\sigma_{D_k}\]
  • Dk: distances of samples assigned to cluster k
  • mu_Dk, sigma_Dk: mean/std for cluster k

Use: Per-cluster dynamic detection threshold.

File: client/clustering.py

F14. Final prediction rule

\[\hat{y}=\begin{cases}1,& d>\theta\\0,& d\le\theta\end{cases}\]
  • d: nearest-centroid distance for incoming sample
  • theta: global or adaptive threshold
  • y_hat: 1 = Attack, 0 = Normal

Use: Convert distance score into class label.

File: client/clustering.py
